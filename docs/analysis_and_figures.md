# Analysis pipeline and figures (quick reference)

Companion to the full specification [data_analysis.md](data_analysis.md). Here: **commands**, **artifacts**, and **figure filenames** only.

---

## Commands

From repo root with `.venv` and `pip install -e '.[analysis]'`:

```bash
.venv/bin/python -m data_analysis.ingest --output-root output
.venv/bin/python -m data_analysis.metrics --output-root output
.venv/bin/python -m data_analysis.plot --output-root output
```

Or one step:

```bash
.venv/bin/python -m data_analysis.pipeline --output-root output
```

Makefile: `make -f scripts/makefile analysis-all` (defaults `--output-root output`).

---

## Processed artifacts (`output/processed/`)

| File | Role |
|------|------|
| `manifest.parquet` (or `.csv`) | One row per solution JSON; path-derived and JSON-derived columns. |
| `paired_metrics.parquet` / `.csv` | Manifest joined to brute-force reference; `approx_ratio_real`, `approx_ratio_objective`, etc. |
| `summary_by_config.csv` | Aggregates by `(n_cities, solver, formulation, qaoa_depth)` — **simulated annealing excluded**. |
| `energy_curves_agg.parquet` / `.csv` | Per-step quantiles/stats of `energy_history` — **SA excluded**. |

Optional: `metrics` with `--sample-quality` adds `final_sample_feasible_mass` (slow).

## Figures (`output/images/`)

Generated by [data_analysis/plot.py](../src/data_analysis/plot.py) from non-empty `paired_metrics` and/or `energy_curves_agg`. Names and meaning match [data_analysis.md](data_analysis.md) § Phase 3; summarized below.

### Energy history (from `energy_curves_agg`)

| PNG | Cohort / use |
|-----|----------------|
| `energy_history_mean_cudaq_qubo_vs_tqudo_virtual_n5.png` | CUDA-Q QUBO vs TQUDO virtual, `n=5`; twin Y axes (scales not comparable across sides). |
| `energy_history_mean_cirq_tqudo_vs_cudaq_tvirt_n5.png` | Cirq native TQUDO vs CUDA-Q virtual, `n=5`; single Y scale. |
| `energy_history_mean_cirq_tqudo_by_ncities.png` | Cirq TQUDO only; one series per `n_cities`. |

Dashed lines reference brute-force **objective** in the same formulation (not raw tour cost).

### Benchmarks and ratios (from `paired_metrics`)

| PNG | Content |
|-----|---------|
| `comparison_cudaq_qubo_vs_tqudo_virtual_by_qaoa_depth.png` | 2×2 dashboard: QUBO vs TQUDO virtual (counts, feasibility, asymmetry by QAOA depth `p`). |
| `comparison_cudaq_tqudo_virtual_vs_cirq_tqudo_by_qaoa_depth.png` | Same layout: CUDA-Q virtual vs Cirq TQUDO. |
| `comparison_mean_approx_ratio_cudaq_qubo_cudaq_tvirt_cirq_tqudo_n5_by_qaoa_depth.png` | Mean \(\rho =\) `approx_ratio_real` vs `p` for three series (feasible rows). |
| `comparison_mean_approx_ratio_cirq_tqudo_n5_n8_cudaq_tvirt_n9_by_ncities.png` | Mean \(\rho\) vs `n` (convention: Cirq up to `n=8`, CUDA-Q virtual at `n=9`). |
| `comparison_cudaq_qubo_vs_tqudo_virtual_opt_steps_both_optimal_by_qaoa_depth.png` | Steps until first global minimum of `energy_history`, pairs optimal on both sides. |
| `comparison_cudaq_tqudo_virtual_vs_cirq_tqudo_opt_steps_both_optimal_by_qaoa_depth.png` | Same for CUDA-Q virtual vs Cirq TQUDO. |

**Note:** Aggregated summaries and energy curves **drop simulated annealing** rows by design; the manifest may still list SA runs.
